Assignment 1 – Market Gap Pitch

Due: Week 3 Weight: 20%

Objective: Identify a market opportunity and design a viable e‑commerce business concept, presented via a professional pitch deck.

Tasks

  • Research trends using tools such as Google Trends, Statista, and SimilarWeb.
  • Fill a Lean Canvas for your idea (Problem, Solution, UVP, Customer Segments, Channels, Revenue, Costs, Key Metrics, Unfair Advantage).
  • Create a 5–6 slide pitch deck covering: Problem, Target Audience, Business Model, Value Proposition, Monetization.
  • Deliver a 5‑minute in‑class presentation.

Deliverables

  • Pitch deck (PDF or PPT).
  • 2‑page business summary (PDF/Doc).

Evaluation (20%)

CriterionWeightDescription
Originality & Feasibility40%Clear market gap; practical go‑to‑market.
Market Research20%Evidence & data‑driven rationale.
Communication20%Clarity, visuals, time management.
Q&A / Defense20%Confident responses and reasoning.

Assignment 2 – No‑code Prototype (MVP)

Due: Week 5 Weight: 20%

Objective: Validate your concept quickly using a no‑code platform.

Tasks

  • Choose a platform (e.g., Shopify, Wix, Webflow, or Bubble).
  • Build an MVP including: Home page & branding, Product listing, Checkout flow (mock or test mode).
  • Conduct a peer usability test; record key findings.
  • Write a short reflection: what worked, what didn’t, and planned improvements for the coded build.

Deliverables

  • Public/live MVP link.
  • 2–3 page usability reflection with screenshots.

Evaluation (20%)

CriterionWeightDescription
Functionality & Design40%Coherent flows; accessible UI; consistent brand.
Usability20%Navigation clarity; feedback‑driven fixes.
Creativity20%Compelling value proposition & messaging.
Reflection Quality20%Honest learnings; clear next steps.

Assignment 3 – Database & SQL Analytics

Due: Week 8 Weight: 20%

Objective: Design and analyze an e‑commerce database in MSSQL.

Schema guidance: include Products, Categories/Tags, Users, Orders, OrderDetails, Payments, Addresses. Add audit fields (createdAt/updatedAt) and sensible indexes.

Tasks

  1. Design an ERD & implement schema in MSSQL with sample data.
  2. Write analytics queries:
    • Customers who signed up but never purchased.
    • Top customers by spend & frequency.
    • Abandoned baskets (open carts without orders).
    • Top‑selling items & categories.
    • Low‑stock items (threshold you define).
    • Monthly sales (revenue, AOV, orders).
  3. Summarize business insights in a concise report.

Deliverables

  • ERD (PDF or image).
  • SQL script (schema + seed + queries).
  • 2‑page insights report.

Evaluation (20%)

CriterionWeightDescription
Schema Quality30%Proper normalization, keys, indexes.
Query Accuracy & Depth40%Correctness; non‑trivial analytics.
Insights20%Actionable takeaways for the business.
Documentation10%Readable scripts; clear assumptions.

Assignment 4 – Full‑Stack Development Project

Due: Week 14 Weight: 40%

Objective: Build a working e‑commerce prototype using MSSQL + Node.js/Express + a modern frontend (e.g., React/Next.js).

Backend (Node.js + Express)

  • CRUD for products, users, orders; input validation.
  • Authentication with JWT (password hashing).
  • Checkout endpoint with Stripe test mode.
  • API docs (Postman collection + README).

Database (MSSQL)

  • Implemented schema & seed data.
  • Views or stored procedures for common reads.
  • Indexes for performance on listing & analytics.

Frontend (React/Next.js or similar)

  • Product list & detail pages.
  • Cart & checkout flow.
  • Login/register forms; basic profile page.
  • Responsive design.

Deployment

  • Host API and frontend (Azure/Render/Heroku/Vercel).
  • Environment variables for secrets; test accounts.
  • Provide live demo links.

Deliverables

  • GitHub repo (with clear README & setup steps).
  • Live deployment link(s).
  • Documentation: ERD, API routes, screenshots.
  • Final presentation deck (story: Pitch → MVP → Full‑stack).

Evaluation (40%)

CriterionWeightDescription
Technical Execution40%Code quality, correctness, security, tests (if any).
Functionality & UX25%End‑to‑end flow works; responsive and usable.
Business Alignment15%Implements core value proposition from pitch.
Presentation & Teamwork20%Clear demo, roles, collaboration evidence.

Summary of Deliverables

  • Assignment 1: Pitch deck + 2‑page summary.
  • Assignment 2: No‑code MVP link + usability reflection.
  • Assignment 3: ERD + SQL script + 2‑page insights report.
  • Assignment 4: Full‑stack app (repo + live link) + final deck.

Tip: keep all assets in a single GitHub organization or Drive folder for easy review.
