Course Description

This course takes a hands-on approach to e-commerce. Students progress from business ideation & pitching, to a no‑code MVP, and finally build a full‑stack e‑commerce application using MSSQL, Node.js (Express), and a modern frontend. The structure mirrors a real startup journey from idea to deployment.

Learning Outcomes

  • Identify market gaps and develop a viable e-commerce business model and pitch deck.
  • Create an MVP using no-code tools (e.g., Shopify, Wix, Webflow) and gather feedback.
  • Design an MSSQL schema and implement relational models for products, orders, and users.
  • Build REST APIs with Node.js + Express; implement authentication and validations.
  • Develop a functional frontend, connect it to APIs, and integrate Stripe (test mode).
  • Deploy to the cloud (Azure/Heroku), document, and present a working product.

9 Node.js & Express API CRUD endpoints for products, users, orders (Postman collection)
10 Auth & Payments JWT login + Stripe test checkout integration
11 Frontend (React/Next.js) Connect pages to APIs; cart & checkout flow
12 Cloud & Deployment Deploy API & frontend (Azure/Heroku); env & scaling basics
13 Project Integration End-to-end integration; bug fixing; documentation
14 Final Presentations Demo journey: Pitch → No‑code MVP → Full‑stack app

Policies

Academic Integrity

All submitted work must be original. Plagiarism or unauthorized code sharing will result in a zero and may trigger disciplinary action. Group work is allowed only when explicitly stated.

Attendance & Participation

Attendance is essential for labs and live demos. More than two unexcused absences may impact the participation component.

Late Work

Late submissions lose 10% per day (up to 3 days). After 3 days, submissions are not accepted unless prior arrangements are made.

Teamwork

Projects may be completed individually or in teams of up to 3. Each team must maintain a shared GitHub repo with meaningful commit history per member.
